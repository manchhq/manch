# Design: UI + Server + Polyglot Monorepo Scaffold

**Date:** 2026-06-15
**Status:** Approved (design phase)

## Goal

Extend Manch from a Rust-only Cargo workspace into a **polyglot monorepo** that
hosts a shared React UI, a desktop (Tauri) shell, and a Rust ConnectRPC server —
without violating Manch's "library-centric, domain-free" stance. This pass is
**scaffolding only**: it establishes structure, tooling, and an end-to-end
"hello version" path. Real agent/prompt streaming is explicitly out of scope.

It also does two small housekeeping tasks: add proper dual-license files and
remove the License section from the README.

## Scope

### In scope
- Dual-license files (`LICENSE-MIT`, `LICENSE-APACHE`); remove README License section.
- pnpm workspace + Turborepo coexisting with the existing Cargo workspace.
- `proto/` as the single wire-contract source feeding both Rust and TS codegen.
- `apps/server` — `connectrpc` + `axum` server with a stub RPC (crate `manch-server`).
- `packages/api` — generated TS Connect client + Connect-Query hooks.
- `packages/ui` — dumb/presentational React components (Tailwind + DaisyUI),
  tested with Vitest + React Testing Library, documented in Storybook.
- `apps/desktop` — Tauri 2 app wiring Connect client → TanStack Router/Query →
  `@manch/ui`.

### Out of scope (deferred)
- The real streaming `Prompt` RPC and any agent/tool/memory wiring (that is the
  README's "first milestone").
- `manch-core` itself (does not exist yet; the server embeds a stub).
- A standalone web app (`apps/web`) — shared UI package only for now.
- Bundling the server as a Tauri sidecar for production (noted as follow-up; dev
  runs the server as a separate process).

## Architecture / Layout

```
manch/
├── Cargo.toml                 # workspace — add manch-server + apps/desktop/src-tauri
├── package.json               # pnpm root; scripts run via turbo
├── pnpm-workspace.yaml        # packages: ["apps/*", "packages/*"]
├── turbo.json                 # task pipeline: build, test, lint, storybook
├── buf.yaml  buf.gen.yaml     # one codegen config → Rust + TS
├── proto/manch/v1/*.proto     # the wire contract (single source of truth)
├── LICENSE-MIT  LICENSE-APACHE
│
├── crates/                    # reusable library substrate (embed these)
│   └── manch-protocol/        # (existing)
│
├── packages/
│   ├── api/                   # generated TS Connect client + connect-query hooks
│   └── ui/                    # dumb React components (Tailwind+DaisyUI, Vitest+RTL, Storybook)
│
└── apps/                      # deployable applications / consumers (run these)
    ├── server/                # NEW: connectrpc + axum, embeds core (stub for now); crate `manch-server`
    └── desktop/               # Tauri 2: Vite+React frontend + src-tauri/ (Rust)
```

> **Placement rationale:** `crates/` holds reusable library substrate you *embed*
> (`manch-protocol`, later `manch-core`/`manch-acp`/`manch-memory`). `apps/` holds
> deployable *consumers* you *run* — the desktop app and the self-hostable server
> alike. The server is a deployable product, not a library, so it lives in `apps/`
> next to the desktop app (matching the README's consumer-vs-library taxonomy).

### Tooling decisions
- **Turborepo** orchestrates JS/TS tasks; `turbo.json` defines `build`, `test`,
  `lint`, `storybook`. Root `package.json` scripts delegate to `turbo run …`.
- **Internal packages export TS source** (no per-package build step). The
  `apps/desktop` Vite bundler compiles `@manch/ui` and `@manch/api` directly.
- **`src-tauri` is a Cargo workspace member** — one `cargo` graph; standard
  Tauri 2 layout.
- **Codegen is gitignored**, with one exception:
  - Rust Connect code: generated by `connectrpc-build` in `manch-server/build.rs`
    (lands under `target/`, never committed).
  - TS Connect code: `buf generate` into `packages/api/src/gen/` (gitignored).
  - **Exception:** TanStack Router's `routeTree.gen.ts` (file-based routing) **is
    committed** — required at typecheck/build time, friction if absent.

## Components

### `proto/manch/v1/manch.proto`
Minimal scaffold service to prove the round trip:
```protobuf
service ManchService {
  rpc GetVersion(GetVersionRequest) returns (GetVersionResponse);
}
```
The streaming `Prompt` RPC is intentionally absent (deferred milestone).

### `apps/server` (crate `manch-server`)
- Deps: `connectrpc` (axum feature), `buffa`, `axum`, `tokio`, `manch-protocol`.
- `build.rs` runs `connectrpc-build` over `../../proto`.
- A stub `ManchService` impl returning the workspace version string.
- Binary serves Connect/gRPC-web on a configurable localhost port (default e.g.
  `127.0.0.1:8787`).

### `packages/api` (`@manch/api`)
- `buf generate` with `@bufbuild/protoc-gen-es` + `@connectrpc/protoc-gen-connect-query`.
- Exports a configured `createConnectTransport` factory, the generated client,
  and Connect-Query hooks. Remeda / Valibot helpers for shaping live here.

### `packages/ui` (`@manch/ui`)
- Purely presentational: components take **props + callbacks only**, no
  data-fetching, no Connect/Query/Router awareness.
- Tailwind + DaisyUI styling.
- Vitest + `@testing-library/react` + jsdom.
- Storybook (react-vite builder).
- Ships a sample `Button` with a story and a passing test to prove the pipeline.

### `apps/desktop` (the "smart" parent)
- Tauri 2: Vite + React frontend, `src-tauri/` Rust shell.
- Owns TanStack **Router** (file-based) + **Query**, the `@manch/api` Connect
  client, `react-hookz` for local state.
- Fetches data and passes it down into `@manch/ui` dumb components.
- A landing route calls `GetVersion` and renders the server version via a
  `@manch/ui` component — the end-to-end proof.

## Data flow

```
@manch/ui (dumb)  ←props/callbacks←  apps/desktop (smart)
                                          ↑
                                   @manch/api (Connect-Query)
                                          ↑
                                   manch-server (Connect / Rust)
```

## Error handling
- Server: Connect error codes from the stub impl (scaffold returns `ok`).
- Client: Connect-Query surfaces transport/RPC errors to TanStack Query; the
  desktop landing route renders a simple error state via a `@manch/ui` component.
- This is scaffold-level; richer error taxonomy arrives with the real RPCs.

## Testing
- `packages/ui`: Vitest + RTL unit tests; sample `Button` test must pass.
- `cargo build` / `cargo test` for `manch-server` (compiles + stub test).
- Turbo `test` task aggregates JS/TS tests.

## Verification target
- `pnpm install` resolves the workspace.
- `turbo run test` → UI tests green.
- `turbo run storybook` (or `pnpm storybook`) renders the sample component.
- `cargo build` → server compiles; `cargo run -p manch-server` serves.
- `pnpm --filter desktop tauri dev` (with server running) shows the server
  version in the Tauri window.

## Build sequence
1. **Licensing** — add `LICENSE-MIT` + `LICENSE-APACHE`; delete README License section.
2. **Workspace scaffold** — pnpm root, `pnpm-workspace.yaml`, `turbo.json`, `proto/`, buf config, `.gitignore` updates.
3. **`packages/ui`** — Tailwind + DaisyUI + Vitest + RTL + Storybook + sample component.
4. **`packages/api` + `apps/server`** — proto codegen both sides, stub RPC.
5. **`apps/desktop`** — Tauri shell wiring it all together.
